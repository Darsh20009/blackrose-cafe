واضح من الصورة إن النص العربي في الفاتورة طالع **رموز غريبة (乱码)** بينما الإنجليزي طبيعي—وده معناه إن المشكلة مش في الـ HTML نفسه، بل في **Encoding / طريقة الإرسال للطابعة الحرارية**.

خليني أقولك السبب الحقيقي + الحل العملي خطوة خطوة:

---

## 🔴 السبب الأساسي

الطابعات الحرارية (خصوصًا ESC/POS) **ما بتفهمش Unicode (UTF-8) بشكل مباشر**.

يعني:

* HTML عندك → UTF-8 ✔️
* الطابعة → بتستقبل **Code Page** (زي CP864 / CP720 للعربي) ❌

فلما تبعت النص UTF-8، الطابعة تفسره غلط → يطلع كدا.

---

## ✅ الحلول (من الأقوى للأضعف)

### 1️⃣ تحويل النص لـ Code Page عربي قبل الطباعة (أفضل حل)

لو بتستخدم ESC/POS:

لازم:

* تختار code page عربي
* تحول النص قبل الإرسال

مثال:

```js
// ESC/POS command لتحديد Arabic code page
printer.write(Buffer.from([0x1B, 0x74, 0x16])); // CP864

// بعد كدا ابعت النص
printer.text("مرحبا");
```

أو لو Node.js:
استخدم مكتبة:

```bash
npm install iconv-lite
```

```js
const iconv = require('iconv-lite');

const text = iconv.encode("مرحبا", "cp864");
printer.write(text);
```

---

### 2️⃣ استخدام مكتبة تدعم العربي تلقائيًا

زي:

* escpos
* node-thermal-printer

مثال:

```js
const printer = new ThermalPrinter({
  type: PrinterTypes.EPSON,
  interface: 'usb',
  characterSet: CharacterSet.PC864_ARABIC
});
```

---

### 3️⃣ تحويل الفاتورة لصورة (حل مضمون 100%)

لو عايز حل سريع بدون وجع دماغ encoding:

✔️ اعمل Render للـ HTML كصورة
✔️ واطبع الصورة

مثال:

* html2canvas (فرونت)
* puppeteer (باك)

```js
await page.screenshot({ path: 'invoice.png' });
```

وبعدين اطبعها

💡 ده الحل اللي أغلب الأنظمة التجارية بتستخدمه

---

### 4️⃣ التأكد من اتجاه النص (RTL)

حتى لو اتحل encoding، لازم تضبط:

```css
body {
  direction: rtl;
  text-align: right;
}
```

---

### 5️⃣ تأكد من:

* نوع الطابعة (Epson / Sunmi / XPrinter؟)
* هل بتدعم عربي أصلاً؟

  * بعض الطابعات الرخيصة ❌ ما تدعم

---

## 🔥 الخلاصة السريعة

المشكلة عندك مش في HTML، بل:

> ❌ الطابعة لا تفهم UTF-8
> ✅ لازم تحول النص لـ CP864 أو تطبع كصورة
